# Plan: Fix TransformerError + Túi đồ lỗi

## Vấn đề 1: `TransformerError: Failed to convert miccuangoc to Member`

**Nguyên nhân:** Các lệnh `/give` trong `cogs/give.py` dùng tham số `nguoi_dung: discord.Member`. Khi admin gõ tên "miccuangoc" thay vì tag user từ autocomplete, Discord không thể resolve thành `Member` (user có thể đã rời server hoặc bot không có quyền đọc member). Lỗi này không được xử lý nên hiện "Unhandled tree error" trong log.

**Fix:** Thêm error handler cho `give_group` để bắt `TransformerError` và trả về message thân thiện thay vì crash.

### File: `cogs/give.py`
- Sửa `_give_error` handler (line 662-668) để bắt `app_commands.TransformerError` → hiện message "Không tìm thấy người chơi này trong server. Hãy tag họ từ dropdown thay vì gõ tên."

## Vấn đề 2: Túi đồ 1 số user không mở được

**Nguyên nhân:** Có 2 khả năng:
1. **View timeout**: `HoSoView` có `timeout=300` (5 phút). Sau 5 phút, tất cả button停止 hoạt động → user bấm vào hiện "This interaction failed".
2. **`_cb_kho_do` thiếu null check**: Nếu `ts` = None (profile bị xóa hoặc lỗi DB), `_build_inventory(None)` sẽ crash vì truy cập `ts["dan_duoc"]`.

### File: `cogs/hoso.py`
- Thêm null check trong `_cb_kho_do` (line 233-245): nếu `ts` là None → hiện thông báo lỗi thay vì crash
- Tương tự thêm null check cho các callback khác (_cb_tu_luyen, _cb_chi_tiet, etc.) để tránh crash khi profile không tồn tại
